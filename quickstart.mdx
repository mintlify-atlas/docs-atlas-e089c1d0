## Overview

In this quickstart, you'll:

1. Create a server-side auth instance
2. Set up a client-side auth client
3. Add authentication API routes
4. Implement sign-up and sign-in

Let's get started!

## Step 1: Create Auth Instance

Create an `auth.ts` file in your project (typically in `lib/` or `src/`):

<CodeGroup>

```typescript auth.ts (Kysely)
import { betterAuth } from "better-auth";
import { PostgresDialect } from "kysely";
import { Pool } from "pg";

export const auth = betterAuth({
  database: new PostgresDialect({
    pool: new Pool({
      connectionString: process.env.DATABASE_URL,
    }),
  }),
  emailAndPassword: {
    enabled: true,
  },
});
```

```typescript auth.ts (Prisma)
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

export const auth = betterAuth({
  database: prismaAdapter(prisma, {
    provider: "postgresql", // or "mysql", "sqlite"
  }),
  emailAndPassword: {
    enabled: true,
  },
});
```

```typescript auth.ts (Drizzle)
import { betterAuth } from "better-auth";
import { drizzleAdapter } from "better-auth/adapters/drizzle";
import { drizzle } from "drizzle-orm/node-postgres";
import { Pool } from "pg";

const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});

const db = drizzle(pool);

export const auth = betterAuth({
  database: drizzleAdapter(db, {
    provider: "pg", // or "mysql", "sqlite"
  }),
  emailAndPassword: {
    enabled: true,
  },
});
```

```typescript auth.ts (MongoDB)
import { betterAuth } from "better-auth";
import { mongodbAdapter } from "better-auth/adapters/mongodb";
import { MongoClient } from "mongodb";

const client = new MongoClient(process.env.DATABASE_URL!);
const db = client.db();

export const auth = betterAuth({
  database: mongodbAdapter(db),
  emailAndPassword: {
    enabled: true,
  },
});
```

</CodeGroup>

<Note>
  The example above shows email/password authentication. Better Auth supports [many authentication methods](/introduction#authentication-methods).
</Note>

## Step 2: Add API Routes

Better Auth needs API endpoints to handle authentication requests. The setup varies by framework:

<Tabs>
  <Tab title="Next.js (App Router)">
    Create `app/api/auth/[...all]/route.ts`:

    ```typescript app/api/auth/[...all]/route.ts
    import { auth } from "@/lib/auth"; // path to your auth.ts
    import { toNextJsHandler } from "better-auth/next-js";

    export const { GET, POST } = toNextJsHandler(auth);
    ```

    <Note>
      The `[...all]` catch-all route handles all Better Auth endpoints like `/api/auth/sign-in`, `/api/auth/sign-up`, etc.
    </Note>
  </Tab>
  
  <Tab title="Next.js (Pages Router)">
    Create `pages/api/auth/[...all].ts`:

    ```typescript pages/api/auth/[...all].ts
    import { auth } from "@/lib/auth"; // path to your auth.ts
    import { toNextJsHandler } from "better-auth/next-js";

    export default toNextJsHandler(auth);
    ```
  </Tab>

  <Tab title="SvelteKit">
    Create `src/routes/api/auth/[...all]/+server.ts`:

    ```typescript src/routes/api/auth/[...all]/+server.ts
    import { auth } from "$lib/auth"; // path to your auth.ts
    import { svelteKitHandler } from "better-auth/svelte-kit";

    export const { GET, POST } = svelteKitHandler(auth);
    ```
  </Tab>

  <Tab title="Solid Start">
    Create `src/routes/api/auth/[...all].ts`:

    ```typescript src/routes/api/auth/[...all].ts
    import { auth } from "~/lib/auth"; // path to your auth.ts
    import { solidStartHandler } from "better-auth/solid-start";

    export const { GET, POST } = solidStartHandler(auth);
    ```
  </Tab>

  <Tab title="Node.js / Express">
    ```typescript server.ts
    import express from "express";
    import { auth } from "./auth"; // path to your auth.ts
    import { toNodeHandler } from "better-auth/node";

    const app = express();

    app.all("/api/auth/*", toNodeHandler(auth));

    app.listen(3000, () => {
      console.log("Server running on http://localhost:3000");
    });
    ```
  </Tab>
</Tabs>

## Step 3: Create Auth Client

Create an `auth-client.ts` file for your frontend:

<CodeGroup>

```typescript auth-client.ts (Vanilla)
import { createAuthClient } from "better-auth/client";
import type { auth } from "./auth"; // Import type from your server auth

export const authClient = createAuthClient({
  baseURL: "http://localhost:3000", // Your app's URL
});

// Infer types from server
export type Session = typeof authClient.$Infer.Session;
```

```typescript auth-client.ts (React)
import { createAuthClient } from "better-auth/react";
import type { auth } from "./auth"; // Import type from your server auth

export const authClient = createAuthClient({
  baseURL: "http://localhost:3000", // Your app's URL
});

export type Session = typeof authClient.$Infer.Session;
```

```typescript auth-client.ts (Vue)
import { createAuthClient } from "better-auth/vue";

export const authClient = createAuthClient({
  baseURL: "http://localhost:3000", // Your app's URL
});

```

```typescript auth-client.ts (Svelte)
import { createAuthClient } from "better-auth/svelte";
import type { auth } from "./auth"; // Import type from your server auth

export const authClient = createAuthClient({
  baseURL: "http://localhost:3000", // Your app's URL
});

```

```typescript auth-client.ts (Solid)
import { createAuthClient } from "better-auth/solid";
```

</CodeGroup>

## Step 4: Implement Authentication UI

Now let's create sign-up and sign-in forms:

<Tabs>
  <Tab title="React / Next.js">
    ```tsx components/auth-form.tsx
    "use client"; // For Next.js App Router

    import { authClient } from "@/lib/auth-client";
    import { useState } from "react";

    export function AuthForm() {
      const [isSignUp, setIsSignUp] = useState(false);
      const [email, setEmail] = useState("");
      const [password, setPassword] = useState("");
      const [name, setName] = useState("");

      const handleSubmit = async (e: React.FormEvent) => {
        e.preventDefault();

        if (isSignUp) {
          await authClient.signUp.email({
            email,
            password,
            name,
          });
        } else {
          await authClient.signIn.email({
            email,
            password,
          });
        }
      };

      return (
        <form onSubmit={handleSubmit}>
          <h2>{isSignUp ? "Sign Up" : "Sign In"}</h2>
          
          {isSignUp && (
            <input
              type="text"
              placeholder="Name"
              value={name}
              onChange={(e) => setName(e.target.value)}
              required
            />
          )}
          
          <input
            type="email"
            placeholder="Email"
            value={email}
            onChange={(e) => setEmail(e.target.value)}
            required
          />
          
          <input
            type="password"
            placeholder="Password"
            value={password}
            onChange={(e) => setPassword(e.target.value)}
            required
          />
          
          <button type="submit">
            {isSignUp ? "Sign Up" : "Sign In"}
          </button>
          
          <button type="button" onClick={() => setIsSignUp(!isSignUp)}>
            {isSignUp ? "Already have an account?" : "Need an account?"}
          </button>
        </form>
      );
    }
    ```

    Display the current session:

    ```tsx components/user-info.tsx
    "use client";

    import { authClient } from "@/lib/auth-client";

    export function UserInfo() {
      const session = authClient.useSession();

      if (session.isPending) {
        return <div>Loading...</div>;
      }

      if (!session.data) {
        return <div>Not signed in</div>;
      }

      return (
        <div>
          <p>Signed in as {session.data.user.email}</p>
          <button onClick={() => authClient.signOut()}>
            Sign Out
          </button>
        </div>
      );
    }
    ```
  </Tab>

## Common Next Features

### Password Reset

Add password reset functionality:

```typescript
// Request password reset
await authClient.forgetPassword({
  email: "user@example.com",
  redirectTo: "/reset-password",
});

// Reset password with token
await authClient.resetPassword({
  newPassword: "newSecurePassword",
});
```

Learn more in the [Password Reset guide](/guides/password-reset).

### Social Authentication

Add OAuth providers to your auth configuration:

```typescript auth.ts
import { betterAuth } from "better-auth";
import { google, github } from "better-auth/social-providers";

export const auth = betterAuth({
  // ... database config
  socialProviders: {
    google: {
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    },
    github: {
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    },
  },
});
```

Learn more in the [Social Providers guide](/authentication/social-providers).

### Add Plugins

Extend functionality with plugins:

```typescript auth.ts
import { betterAuth } from "better-auth";
import { twoFactor } from "better-auth/plugins";

export const auth = betterAuth({
  // ... other config
  plugins: [
    twoFactor({
      issuer: "MyApp",
    }),
  ],
});
```

Explore all [available plugins](/plugins/overview).

## Troubleshooting

<AccordionGroup>
  <Accordion title="Session not persisting after sign-in">
    Ensure cookies are being set correctly. Check:
    - `baseURL` in your client matches your server URL
    - CORS is configured if frontend and backend are on different domains
    - Cookies are not being blocked by browser settings
  </Accordion>

  <Accordion title="Type errors with session">
    Make sure you're importing the type from your server auth:
    
    ```typescript
    import type { auth } from "./auth";
    ```

    The client will automatically infer types from your server configuration.
  </Accordion>

  <Accordion title="Database connection errors">
    Verify:
    - `DATABASE_URL` is set correctly in `.env`
    - Database migrations have been run: `npx @better-auth/cli migrate`
    - Database server is running and accessible
  </Accordion>

  <Accordion title="API routes not found (404)">
    Check:
    - API route file is in the correct location for your framework
    - File exports the correct handlers (GET, POST)
    - Dev server has been restarted after creating routes
  </Accordion>
</AccordionGroup>
