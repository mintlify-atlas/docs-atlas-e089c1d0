## Overview

Email and password authentication is the most traditional authentication method. Better Auth provides a secure, production-ready implementation with built-in password hashing, validation, and session management.

## Features

- Secure password hashing with configurable algorithms
- Password strength validation
- Email verification support
- Password reset flow
- Auto sign-in on registration
- Custom password requirements

## Installation

Email and password authentication is built into Better Auth core - no additional packages required.

## Configuration

### Server Setup

<Steps>

<Step title="Enable email and password">

Configure email and password authentication in your auth configuration:

```ts title="auth.ts"
import { betterAuth } from "better-auth"

export const auth = betterAuth({
  database: {
    // your database configuration
  },
  emailAndPassword: {
    enabled: true,
    minPasswordLength: 8,
    maxPasswordLength: 128,
    requireEmailVerification: false,
    autoSignIn: true,
  },
})
```

</Step>

<Step title="Configure email verification (optional)">

Set up email verification for enhanced security:

```ts title="auth.ts"
import { betterAuth } from "better-auth"

export const auth = betterAuth({
  emailAndPassword: {
    enabled: true,
    requireEmailVerification: true,
  },
  emailVerification: {
    sendOnSignUp: true,
    sendVerificationEmail: async ({ user, url, token }) => {
      // Send verification email
      await sendEmail({
        to: user.email,
        subject: "Verify your email",
        html: `Click <a href="${url}">here</a> to verify your email.`,
      })
    },
  },
})
```

</Step>

<Step title="Set up password reset (optional)">

Enable password reset functionality:

```ts title="auth.ts"
import { betterAuth } from "better-auth"

export const auth = betterAuth({
  emailAndPassword: {
    enabled: true,
    sendResetPassword: async ({ user, url, token }) => {
      // Send password reset email
      await sendEmail({
        to: user.email,
        subject: "Reset your password",
        html: `Click <a href="${url}">here</a> to reset your password.`,
      })
    },
    resetPasswordTokenExpiresIn: 3600, // 1 hour in seconds
  },
})
```

</Step>

</Steps>

## Usage

### Sign Up

<Tabs items={["React", "Vue", "Svelte", "Vanilla"]}>

<Tab value="React">

```tsx title="components/SignUpForm.tsx"
import { signUp } from "@/lib/auth-client"
import { useState } from "react"

export function SignUpForm() {
  const [email, setEmail] = useState("")
  const [password, setPassword] = useState("")
  const [name, setName] = useState("")

  const handleSignUp = async (e: React.FormEvent) => {
    e.preventDefault()
    
    await signUp.email({
      email,
      password,
      name,
      callbackURL: "/dashboard",
    })
  }

  return (
    <form onSubmit={handleSignUp}>
      <input
        type="text"
        value={name}
        onChange={(e) => setName(e.target.value)}
        placeholder="Name"
        required
      />
      <input
        type="email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Email"
        required
      />
      <input
        type="password"
        value={password}
        onChange={(e) => setPassword(e.target.value)}
        placeholder="Password"
        required
      />
      <button type="submit">Sign Up</button>
    </form>
  )
}
```

## Configuration Options

### Email and Password Options

```ts
interface EmailAndPasswordOptions {
  /**
   * Enable email and password authentication
   * @default false
   */
  enabled: boolean

  /**
   * Minimum password length
   * @default 8
   */
  minPasswordLength?: number

  /**
   * Maximum password length
   * @default 128
   */
  maxPasswordLength?: number

  /**
   * Require email verification before sign in
   * @default false
   */
  requireEmailVerification?: boolean

  /**
   * Auto sign in after registration
   * @default true
   */
  autoSignIn?: boolean

  /**
   * Disable sign up
   * @default false
   */
  disableSignUp?: boolean

  /**
   * Send password reset email function
   */
  sendResetPassword?: (data: {
    user: User
    url: string
    token: string
  }, request: Request) => Promise<void>

  /**
   * Password reset token expiration in seconds
   * @default 3600 (1 hour)
   */
  resetPasswordTokenExpiresIn?: number

  /**
   * Revoke all sessions on password reset
   * @default false
   */
  revokeSessionsOnPasswordReset?: boolean

  /**
   * Callback when password is reset
   */
  onPasswordReset?: (data: {
    user: User
  }, request: Request) => Promise<void>
}
```

## API Reference

### Server Endpoints

- `POST /sign-up/email` - Register a new user (see auth.ts:27-359)
- `POST /sign-in/email` - Sign in with email and password (see sign-in.ts:335-574)
- `POST /request-password-reset` - Request password reset (see password.ts:36-149)
- `GET /reset-password/:token` - Verify reset token (see password.ts:151-226)
- `POST /reset-password` - Reset password (see password.ts:228-339)
- `POST /verify-password` - Verify current password (see password.ts:341-398)

### Client Methods

```ts
// Sign up
await authClient.signUp.email({
  email: string
  password: string
  name: string
  image?: string
  callbackURL?: string
  rememberMe?: boolean
})

// Sign in
await authClient.signIn.email({
  email: string
  password: string
  callbackURL?: string
  rememberMe?: boolean
})

// Request password reset

  email: string
  redirectTo?: string
})

// Reset password
await authClient.resetPassword({
  newPassword: string
  token: string
})

// Verify password
await authClient.api.verifyPassword({
  password: string
})
```

## Security Features

### Password Hashing

Better Auth uses secure password hashing by default. The hashing is performed before creating the user to ensure security (see sign-up.ts:252).

### Timing Attack Prevention

The sign-in flow includes timing attack prevention by hashing passwords even for non-existent users (see sign-in.ts:466-468).

### Password Validation

Passwords are validated against configured requirements:
- Minimum length check (see sign-up.ts:220-226)
- Maximum length check (see sign-up.ts:228-234)
- Required fields validation (see sign-up.ts:214-218)

## Best Practices

1. **Enable Email Verification**: For production apps, always require email verification
2. **Strong Password Requirements**: Set appropriate `minPasswordLength` (recommended: 12+)
3. **Password Reset Flow**: Always implement password reset for better UX
4. **Session Management**: Use `rememberMe: false` for sensitive operations
5. **Rate Limiting**: Configure rate limits to prevent brute force attacks
